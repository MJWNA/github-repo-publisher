# Contributing

Thanks for considering a contribution.

## Local Setup

```bash
# install dependencies
# run tests
# run lint/build
```

## Workflow

- Create an issue or comment on an existing one before large changes.
- Use a focused branch.
- Keep pull requests small enough to review.
- Add tests or explain why tests are not applicable.
- Update docs when behavior changes.

## Pull Requests

Before opening a PR:

- Tests pass locally.
- Lint/build passes locally.
- Public APIs, docs, and examples are updated.
- Security-sensitive changes are called out clearly.

## Security

Do not report vulnerabilities in public issues. Follow [SECURITY.md](SECURITY.md).
