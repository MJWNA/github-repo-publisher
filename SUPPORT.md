# Support

Use GitHub issues for:

- workflow bugs
- unclear instructions
- README template gaps
- GitHub CLI or API command issues
- suggestions for additional repo profiles

Use `SECURITY.md` for:

- accidental exposure risks
- unsafe GitHub Actions permissions
- secret handling concerns
- branch protection or ruleset issues that could weaken repository security

Do not post secrets, tokens, production URLs, private repository names, customer data, or internal system details in public issues.
