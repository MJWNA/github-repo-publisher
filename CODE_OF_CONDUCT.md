# Code Of Conduct

## Our Standard

Be respectful, constructive, and practical. This project exists to improve repository publishing workflows, so feedback should focus on behavior, documentation quality, safety, and maintainability.

Examples of welcome behavior:

- explaining tradeoffs clearly
- giving specific, actionable feedback
- respecting different experience levels
- documenting security concerns privately when appropriate

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- sharing private information without permission
- posting secrets, credentials, or sensitive operational details
- disruptive or bad-faith participation

## Enforcement

Project maintainers may remove comments, close issues, or block contributors when behavior is harmful to the project or community.

For sensitive concerns, use the security reporting route in `SECURITY.md` when appropriate.
